# Components Baseline

Este documento define o uso comum de Skills, Workflows, Quality e Templates no Framework Nébula.

## Objetivo

1. Reduzir repetição entre manuais de componentes.
2. Definir o fluxo comum de uso entre pilares.
3. Manter rastreabilidade com Docs como fonte de verdade.

## Regra comum

1. Templates são modelo de preenchimento.
2. Artefatos oficiais do projeto são editados em Docs.
3. Docs/Prototype é exclusivo para protótipos HTML.

## Fluxo comum entre componentes

1. Definir objetivo da task e impacto esperado.
2. Selecionar workflow principal da mudança.
3. Carregar skills da especialidade necessária.
4. Atualizar documentos oficiais em Docs.
5. Executar Quality Gate.
6. Registrar evidências em Docs/control.md.
7. Fechar a task com 1 commit.

## Matriz de papéis

1. Skills: orientam como executar tecnicamente.
2. Workflows: orientam ordem e evidências por tipo de mudança.
3. Quality: valida requisitos mínimos de confiabilidade.
4. Templates: aceleram estruturação dos artefatos em Docs.
