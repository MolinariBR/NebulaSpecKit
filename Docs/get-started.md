# Get Started

## Objetivo

Este documento é o ponto de partida da documentação oficial do projeto.

No Nébula, a implementação só começa depois que a base documental está editada em `Docs/`.

## Princípios do fluxo

1. `Docs/` é a fonte de verdade do projeto.
2. `Templates/` é modelo de preenchimento, não destino final.
3. `Docs/Prototype/` guarda apenas protótipos HTML de interface.
4. Cada task fecha com evidência e Quality Gate.

## Ordem recomendada de preenchimento

1. `brief.md`
2. `project.md`
3. `stack.md`
4. `user-stories.md`
5. `pages.md`
6. `flow.md`
7. `design-system.md`
8. `entities.md`
9. `architecture.md`
10. `contract.yaml`
11. `structure.md`
12. `deploy.md`
13. `plan.md`
14. `tasks.md`
15. `control.md`

## Como operar na prática

1. Escolha o template correspondente em `Templates/Full` ou `Templates/Quick`.
2. Use o template para estruturar o conteúdo.
3. Salve e mantenha a versão oficial no arquivo equivalente em `Docs/`.
4. Registre avanço em `control.md` e execute o Quality Gate antes de concluir a task.

## Critério de pronto da fase documental

A fase documental está pronta quando:

1. Há clareza de escopo, contexto e objetivos.
2. Fluxo funcional e impacto técnico estão descritos.
3. Plano, tasks e controle estão atualizados.
4. O time consegue iniciar implementação sem suposições críticas.

## Próximo passo

Com a documentação mínima validada, siga para execução usando:

1. `Docs/plan.md`
2. `Docs/tasks.md`
3. `Docs/control.md`

e os workflows aplicáveis no diretório `Workflows/`.
