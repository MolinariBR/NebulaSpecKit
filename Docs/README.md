# Docs

## Descrição

Ao usar o Nébula Framework, inicialmente deve ser criada toda a documentação de governança, execução e qualidade em `Docs/`.

Essa documentação é a fonte de verdade do projeto e deve ser mantida continuamente atualizada. Ela orienta agentes de execução (Claude Code, GitHub Copilot e outros) e pessoas do time.

Na interface web de documentação, o ponto de entrada é `get-started.md`.

## Regra de uso

1. Templates em `Templates/` são modelos de preenchimento.
2. Artefatos oficiais do projeto são salvos em `Docs/`.
3. Protótipos HTML são salvos em `Prototype/`.
